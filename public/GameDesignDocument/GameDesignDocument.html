AquaCycle

A Game For Web Browsers

Copyright 2017 Freddy Estevez, Devon Maguire, Kwaku Owusu

INTRODUCTION

AquaCycle is a web based game that is designed to teach people more about the life of aquatic animals. AquaCycle will be portrayed from a 2D Top-Down Perspective and will implement sprite animation, as well as tiled backgrounds.

TECHNOLOGY

AquaCycle will be developed for web browsers across all operating system using the Phaser JavaScript Framework. Components such as sounds and rendering will all be handled by Phaser. Our artwork will be created using tools such as GIMP. Our levels will be created using the Tiled tileset editor.

BACKSTORY

AquaCycle follows the life of one of three playable aquatic animals. The player will be tasked with exploring a section of ocean and finding food, and also discovering information about the animal’s habitat. However, not all is as peaceful as it sounds, the player will also have to avoid natural predators and other dangers they encounter as they explore.

OBJECTIVE

The character’s objective in the game is to interact with a certain number of objects in order to move on to the next level while avoiding harmful predators. When interacting with an object, the player will see some information pertaining to the target. Each level may have its own unique set of things to explore.

GAME PLAY

The game will work from a top down perspective. The main character, a creature in the ocean, can move forward by using the ‘W’ key and can use the ‘A’ key to move left and the ‘D’ key to move right while moving forward.

The main character can interact with different animals and objects in the ocean. These fall into three different categories. Predators are animals and objects that can hurt the main character, Prey are animals and objects that can help the main character or things that can be eaten, and Items are objects that can be explored but don’t directly interact with the player. Some predators, if they get within a certain distance of the main character will start to track the player until they move out of the predators range again.

CONTROLS

This game will be played using both a keyboard and a mouse. Once the game has started these are the following controls:

W - The player moves forward.

S - The player moves backwards.

A - The player moves to the left.

D - The player moves to the right.

Click - When a player clicks on an object in the game, information about the object will appear in the on screen information box.

ESC - This pauses the game and presents an in-game pop-up screen that allows a player to save, restart, go back to the home screen, adjust the in-game volume, and see which objects they have explored or interacted with in the game.

GRAPHICAL USER INTERFACE

As far as the GUI is concerned we will need the following:

Splash Screen - The splash screen GUI simply presents a game logo and a login button for the user to press when they are ready to login. Upon pressing it, the user will be redirected to login to enter the Game Start Page.

Navbar - Presents the user with buttons to navigate to the game start, level, control, and help pages. Only shown in the game start, level, control, and help pages.

Game Start Page -  Presents the game logo and a button to start a new game and a button to continue his/her current game.

Levels Page - Presents the three levels of the game, the progress of completion, and if the level is locked/unlocked.

Controls Page -  Illustrates all the accepted inputs during the game and the actions relating to them.

Help Page - Illustrates an image of the 3 characters in the game, a brief description of each, and the developers who created the game.

In-Game Screen - The screen displayed while the player is playing the game. This includes an experience bar, health hearts, and the number of items explored for the level. This screen also has an information box in the lower right to display information on items when found.

In-Game Menu - While the game is in progress a user may press the ESC key to bring up the in-game menu which will depict all the current items found in the level as well as the following options:

  • Save Game -  Save current progress in the level.
  • Home - Prompt the user if they want to save first and then return home.
  • Restart -  Restart the level user is currently on.
  • In-Game Volume Slider - slider for user to change the volume of the game sounds.

All of the artwork in the game will be original work. The following is a list of things that need to be created. For the main characters there will be three different creatures for each level, but all will need the same animations:

Main Character - The characters will have the following animations:

        Hatching

        Swimming

        Taking damage

        Eating

        Evolving

Dying

Background Tiles - Since this is a top down game, the following tiles will need to be created:

        Depth Tiles

                Near Surface (indicated by a light blue)

                Middle Depth (indicated by a blue tone in the middle of light and dark)

                Deep (indicated by a dark blue)

        Item Tiles

                Seaweed

                Coral

                Shells

Collidable Tiles - The following tiles will be created that the player can collide with:

        Shore Tiles

        Stationary Predators

                Trash

Prey - There will be four to six prey per level that the main player can eat or interact with in a beneficial way. They will have the following animations:

        Swimming

Predator - There will be four to five predators per level that will hurt the main player. They will have the following animations:

        Swimming

        Attack
